# 用户旅程对比图

## 场景：周一下午3点，办公室，想喝咖啡

### 传统旅程（现状）

```
用户内心：有点困了，想喝杯咖啡
 ↓
打开微信 → 翻聊天记录找瑞幸小程序
 ↓
打开瑞幸小程序 → 浏览菜单 → 纠结喝什么
 ↓
看价格 → 找优惠券 → 在卡包里翻半天
 ↓
选品 → 加配料(冰/糖) → 下单支付
 ↓
到店 → 排队 → 取餐
 ↓
整个过程：4-6分钟 | 摩擦点：3-4个 | 失败概率：~40%（中途放弃）
```

**用户可能的心理活动：** "算了太麻烦了" "优惠券在哪里" "喝什么好呢" → **跳出流失**

---

### Agent 驱动旅程（本方案）

```
Agent 实时感知：
 ① 时间=周一15:02 → 典型"下午困倦窗口"
 ② 位置=办公区(历史高频消费区) 
 ③ 步数=低(久坐) → 精力下降信号
 ④ 日历=无会议 → 适合下单
 ↓
Agent 主动发起对话（微信消息/企微）：
 「感觉你下午有点困了，还是老样子冰美式去冰？
 我已经用了你的 -¥3.5 下午茶券，¥9.9 搞定，
 你公司楼下店现在不排队，3分钟可取 」
 ↓
用户回复：「好」或直接点击「确认下单」
 ↓
Agent 完成：下单 → 支付 → 发取餐码 → 倒计时提醒
 ↓
下单后 Agent 自动追踪：
 10分钟后：「咖啡好了，趁冰喝哦」
 周五下午：「这周喝了4杯美式了，明天周末要不要试试新上的橙C？
 今天有买一送一，和同事拼单更划算~」
 ↓
整个过程：10-20秒（一句话/一个点击）| 摩擦点：0-1个 | 成功率：~70%
```

---

## 核心对比总结

| 维度 | 传统模式 | Agent 模式 |
|------|---------|-----------|
| 触发方式 | 用户主动搜索(被动) | Agent 预判主动触达(主动) |
| 决策路径 | 打开App→浏览→纠结→下单(6步+) | 一句话确认(1-2步) |
| 优惠券使用 | 手动翻找，经常漏用 | Agent 自动匹配最优组合 |
| 新品触达 | 推送消息，用户忽略 | 场景化自然推荐(如周末推荐) |
| 下单耗时 | 4-6分钟 | 10-20秒 |
| 中途流失率 | ~40% | ~5% |
| 用户感受 | 「又要操作一遍」 | 「瑞幸真懂我」 |

## 全生命周期用户旅程（Agent 视角）

### Stage 1：新用户破冰（第1-3次消费）

```
用户行为：试试看，价格敏感，不确定口味
Agent策略：
 · 首单引导：对话式推荐入门款（"第一次喝推荐生椰拿铁，是瑞幸经典"）
 · 口味采集：自然对话中收集偏好（"冰的还是热的？""甜度习惯？"）
 · 首次复购：第2天下午自动问候（"昨天那杯怎么样？今天试试另一个爆款？"）
```

### Stage 2：习惯养成（第4-15次消费）

```
用户行为：形成基本偏好，开始有记忆的品
Agent策略：
 · 场景建模：记录用户消费的时间/位置/天气模式
 · 周期预测：预判下次消费时间窗口
 · 组合推荐：开始推荐咖啡+轻食组合，提升客单价
```

### Stage 3：忠诚深化（16次+）

```
用户行为：有固定口味，接受推荐
Agent策略：
 · 个性化礼物推荐：朋友生日提醒买咖啡券
 · 新品优先体验：给忠诚用户首发试饮权
 · UGC激励：推荐用户晒单、写评价、获得积分
 · 社交裂变：帮用户组队拼单、邀请同事入群
```

### Stage 4：防流失干预（沉默5天+）

```
用户行为：消费间隔变长，活跃度下降
Agent策略：
 · 轻度唤醒（Day 5-7）："最近忙什么呢？给你带了一杯"
 · 中度唤醒（Day 8-14）：个性化优惠+新品推荐组合
 · 重度召回（Day 15-30）：亲友邀请+大额回归券
 · 沉默止损（Day 30+）：降低触达频次，保存用户画像
```
