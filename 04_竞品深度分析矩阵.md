## 二、AI Agent 竞品对标（公开信息）

| 竞品动态 | 进展阶段 | 与本方案对比 |
|---------|---------|------------|
| 星巴克「Deep Brew」Prediction Engine | 已落地：预测库存、排班、个性化推荐邮件 | 无用户侧Agent，仅后端预测 |
| 麦当劳「动态菜单板」AI | 已落地：根据天气/时段/库存调整推荐 | 门店侧优化，无用户侧对话能力 |
| 美团「智能搜索+推荐」 | 已落地：搜索意图理解+个性化榜单 | 平台逻辑，非品牌自有触达 |
| 微信「企微智能助手」 | 公测中：自动回复+简单任务执行 | 通用工具，无行业深度定制 |
| 支付宝「AI生活管家」 | 内测：对话式服务聚合 | 平台Agent，品牌话语权受限 |

## 三、差异化优势矩阵（本方案 vs 行业现有方案）

| 竞争维度 | 行业现有方案 | 本方案优势 |
|---------|------------|-----------|
| 推荐逻辑 | 基于规则/RFM的静态推荐 | LLM推理+协同过滤混合决策，动态适应 |
| 触达时机 | 固定时间统一推送 | 多模态预判，每个用户独立的"最佳时刻" |
| 交互方式 | 推送信息/短信/模板消息 | 自然对话，双向交流 |
| 优惠策略 | 统一优惠券/等级权益 | 意图触发式智能发券，每人最优组合 |
| 记忆能力 | 基础历史订单记录 | 长期偏好+短期意图的双层记忆网络 |
| 频次控制 | 固定频次限制 | 自适应频次，根据用户状态动态调整 |
| 服务边界 | 仅交易 | 全生命周期：唤醒→推荐→下单→售后→忠诚 |
| 数据资产沉淀 | 订单数据为主 | 互动数据+意图数据+偏好数据的多维度资产 |
